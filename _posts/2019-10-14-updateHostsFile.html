test를 진행하기위한 해당 IP에 도메인을 등록해주는 방법을 알아보자.

hosts파일 수정

해당 사용하고있던 OS가 windows10이었는데, hosts파일의 경로는 다음과 같았다. C:\Windows\System32\drivers\etc\hosts 파일편집기를 이용하여 실행하면 # Copyright (c) 1993-2009 Microsoft Corp. # # This is a sample HOSTS file used by Microsoft TCP/IP for Windows. # # This file contains the mappings of IP addresses to host names. Each # entry should be kept on an individual line. The IP address should # be placed in the first column followed by the corresponding host name. # The IP address and the host name should be separated by at least one # space. # # Additionally, comments (such as these) may be inserted on individual # lines or following the machine name denoted by a '#' symbol. # # For example: # # 102.54.94.97 rhino.acme.com # source server # 38.25.63.10 x.acme.com # x client host # localhost name resolution is handled within DNS itself. # 127.0.0.1 localhost # ::1 localhost 으로 나오는데 현재 사용하는 IP와 사용할 도메인을 적어주면된다. 예를들어 192.168.194.10 www4.testDomain.com 를 작성하고 저장한다면 웹에서 www4.testDomain.com이 열리는것을 확인할 수 있다.

추가정보

네트워크에서 호스트 이름을 IP주소로 변환하는데, DNS보다 hosts파일의 우선순위가 높다고한다. hosts파일은 오래전에 내부망에 자체 DNS 없이 여러 서버들을 사용하는 경우 IP주소를 외우기 힘들기 때문에 서버들에 이름을 붙여서 hosts파일에 등록 해두고 많이 사용했다고 한다. 이는 내부망에서 작업할 일이 있어서 알게된 사실이다.
