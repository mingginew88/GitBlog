앞으로 공부하는 내용을 정리해보고자 Gitblog를 생성하였다. 첫 글은 Gitblog를 생성하는 과정에 대해 작성해보고자 한다.

Github 로그인

Github 계정으로 로그인을 하여야 깃블로그 생성이 가능하므로 로그인을 진행하자. 없다면 계정을 하나 생성해주자.

Repository 설정

좌측 New 라고 적힌 녹색 버튼을 클릭하여 새로운 Repository 생성을 진행하자. New 버튼을 클릭하였다면, 다음과같은 화면이 뜰것이다.

Repository Name 칸에 본인의 계정명.github.io 라고 입력해주면 된다. 평소 Repository를 생성한다면 원하는 이름을 입력하면 되지만, 깃허브 블로그를 만들 때에는 계정명.github.io를 입력하지않으면 안된다. 화면에는 나오지않았지만 스크롤을 내려 아래 create repository 버튼을 눌러 repository를 생성해보자.

jeykll

jeykll은 웹사이트나 블로그를 손쉽게 만드는데 도움을 준다. jeykll에 대해 더 자세히 알아보고자 한다면 공식홈페이지는 : https://jekyllrb-ko.github.io/이다. 템플릿을 적용하여 진행하려하여 구글에 jeykll theme 이라고 검색하여

홈페이지 들어가서 Demo로 확인해보고 원하는 템플릿을 선택하여 다운로드 받는다.

로컬저장소 생성

다음 진행하고자 하는것은 다운로드 받은 파일을 깃허브와 연결하려한다. 사용하고 있는 소스트리를 이용하여 로컬저장소를 생성하고 원격저장소와 연결을 진행한다.

새로만들기를 클릭하여 로컬저장소 생성을 눌러 목적지 경로를 이전에 다운로드 받은 테마폴더로 해준다. 생성한 로컬저장소를 더블클릭하여 접속한다.

우측상단의 설정버튼을 클릭하고, 뜨는 창에 원격 탭을 클릭한다. 추가버튼을 클릭하여 깃허브에 생성한 URL 경로를 추가해준다.

여기까지 진행이후 새 웹사이트를 켜서 계정명.github.io 를 치고 블로그화면을 확인해본다.
