시스템 필수사양 확인

https://help.qlik.com/en-US/sense-admin/November2019/Subsystems/DeployAdministerQSE/Content/Sense_DeployAdminister/Common/system-requirements.htm

설치파일

1. NDP452-KB2901907-x86-x64-AllOS-ENU 2. Qlik_Sense_setup 3. Qlik_Sense_update (업데이트 파일이 있을 시)

방화벽 해제

제어판\시스템 및 보안\Windows Defender 방화벽\설정 사용자 지정 경로에서 개인 네트워크 설정 및 공용 네트워크 설정을 "사용안함"으로 변경

UAC 해제

제어판\사용자 계정\사용자 계정 경로에서 "사용자 계정 컨트롤 설정 변경" 클릭하거나 windows 검색에서 "UAC"로 "사용자 계정 컨트롤 설정 변경" 접근하여 가장 낮음으로 설정 변경

설치 게정 생성 및 권한 부여

windows 검색에서 "netplwiz"로 검색하여 Qlik Sense Server를 위한 계정 생성을 하고 "사용자 이름과 암호를 입력해야 이 컴퓨터를 사용 할 수 있음"을 선택해주고 생성한 사용자를 더블 클릭하여 "그룹 등록" 탭을 눌러 관리자(Administrator) 권한을 부여한다.

설치

Install 버튼 눌러 진행한다.

라이센스 조항에 동의를 하고자 "I accept the license agreement" 항목 선택하여 Next 버튼을 누른다.

새로운 노드를 만들지 기존 노드에 추가할지를 결정하는 것 같은데 "Create cluster" 버튼을 클릭하여 진행한다.

machine 명은 되도록이면 처음 잡아주는 값을 유지하여 진행 하도록 한다.

"Install local database"항목을 선택하고 "Database user password"를 입력하고 동일하게 "Confirm password"를 입력해준다

여러 노드를 쓸 때 IP range에 추가해 주는 거로 알고 있지만 어떤식으로 활용하는지는 아직 모르겠다.

"Username"은 .\ 를 적어주고 그 뒤에 사용하는 컴퓨터 계정과 그에 해당하는 암호도 적어준다.

Root folder를 위해 폴더 경로를 설쟁해 주면 되는데 나의 경우 C:\QlikSense\Shared 로 설정해두었다. 작성 시 \\localhost\ 를 경로 앞에 붙여주고 C: 대신 C$ 로 표시해준다. 정리해보면 최종 Root 경로는 \\localhost\C$\QlikSense\Shared 로 하였다.

Centralized Logging은 logging 방식을 setting해주는거 같은데 아직 활용법을 알지 못하여 "Configure centralized logging"항목을 해제하고 "Next"버튼을 누른다.

설치 경로는 기본 제공해주는 설치 경로로 잡아주었다.

Repository Database Superuser Password 를 작성해준다.

설치 후 서비스를 시작하는 항목 "Start the Qlik Sense services when the setup is complete"을 해제해주고 "Next"를 눌러 설치를 진행한다.

이것으로 Qlik Sense Server 설치를 마친다.
